# Backend — File & Folder Summary

This document describes every file and folder inside the `backend/` directory of the INFLUENCE influencer-marketplace project.

---

## Top-level Files

| File | Purpose |
|------|---------|
| `server.js` | Main entry point — boots Express, connects to MongoDB, initialises Socket.io, Redis, email service, and cron jobs. Registers all route groups and global error handling. |
| `package.json` | npm manifest — lists 71 production dependencies (Express, Mongoose, Socket.io, Stripe, Razorpay, PayPal, Bull, Agenda, Cloudinary, Twilio, Winston, …) and dev tools (Jest, Supertest, MongoDB Memory Server). |
| `package-lock.json` | Exact dependency lockfile generated by npm. |
| `jest.config.js` | Jest test-runner configuration — module resolution, test environment, and coverage settings. |
| `.env` | Local environment variables (never committed to VCS) — DB URI, JWT secrets, payment keys, SMTP, Redis, Cloudinary, Twilio, VAPID, etc. |

---

## Directories

### `config/`
Application-level configuration objects (database connection, cache settings, logger initialisation, environment helpers).

| `brandController.js` | Brand profile CRUD, team management, campaign creation |
| `creatorController.js` | Creator profile, portfolio, verification, analytics |
| `campaignController.js` | Campaign lifecycle management and targeting |
| `dealController.js` | Deal negotiations, status transitions, timeline |
| `paymentController.js` | Payment processing, refunds, invoices |
| `notificationController.js` | Notification CRUD and preference management |
| `chatController.js` | Conversations, messages, delivery/read status |
| `searchController.js` | Advanced creator/campaign search with filters |
| `analyticsController.js` | Campaign ROI, engagement metrics, report generation |
| `affiliateController.js` | Referral programme and commission tracking |
| `featuredController.js` | Featured listing creation and management |
| `complianceController.js` | GDPR data export, consent management |

| `admin/` | Sub-folder with four admin-specific controllers for user management, system reports, payment reconciliation, and content moderation |

### `crons/`
**`cronJobs.js`** — node-cron scheduler that triggers recurring tasks:

| Schedule | Task |
|----------|------|
| Every minute | Deadline proximity alerts |
| Every 5 minutes | Stuck-payment cleanup |
| Hourly | Full deadline checks, payment retry |
| Daily | Analytics generation, token-blacklist cleanup, data purge |
| Weekly | Weekly report dispatch |
| Monthly | Monthly report generation |
| Every 6 hours | Token blacklist maintenance |

---

### `jobs/`
Background job processors powered by Bull/Agenda.

| File | What it processes |
|------|------------------|
| `notificationJobs.js` | Async notification delivery (email, SMS, push) dequeued from Bull |

### `utils/`
Shared helper utilities used across controllers and services (token helpers, date formatters, response builders, validation utilities, etc.).

Runtime log files written by Winston (auto-created, not committed to VCS).

## Key Features at a Glance

- **Creator–Brand marketplace** with an intelligent matching engine
- **Full deal lifecycle** — discovery → negotiation → contract → payment → review
- **Multi-gateway payments** (Stripe, Razorpay, PayPal) with escrow and refund support
- **Subscription tiers** (Free / Professional / Enterprise) with variable commission rates
- **Real-time chat** and notification system (in-app, email, SMS, push)
- **Social media integrations** — Instagram, YouTube, TikTok stats
- **GDPR compliance** — data export, consent tracking, right-to-erasure
- **Analytics & reporting** — ROI, engagement, Excel/PDF exports
- **Security hardening** — 2FA, rate limiting, CSRF/XSS protection, audit logs
- **Admin dashboard** — user moderation, system reports, payment reconciliation
- **Affiliate programme** — referral links and commission tracking
- **Featured listings** — promoted creator and brand slots
